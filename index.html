IntelliJ fast-open plugin

Download

This plugin adds "Open Project Fast [...]" menu item in the "File" menu. This menu opens the project currently selected in the clipboard, and thereby allowing the following workflow that lets you open a project more quickly.

Select a project in Explorer/Nautilus

  1. Select a .ipr file, pom.xml, or a directory that contains either of those in Windows Explorer, GNOME Nautilus, Konquerer, or any other file manager.
  2. Come back to IntelliJ IDEA, type "Alt+F > F" to open that project.

Select a project from anywhere

  1. Send a file name of a .ipr file, pom.xml, or a directory that contains either of those from your editor, your shell, your web browser, your e-mail, or just about anywhere.
  2. Come back to IntelliJ IDEA, type "Alt+F > F" to open that project.

This plugin is particularly effective when you have a small shell script that lets you send arbitrary file name into clipboard. On Unix, such script would look something like: 

#!/bin/sh
D=`dirname "$1"`
B=`basename "$1"`
abspath="`cd \"$D\" 2>/dev/null && pwd || echo \"$D\"`/$B"
echo $abspath | xclip -selection clipboard

On Windows it will look something like this: 

@echo off
echo %~f1 | clipboard -

In this way, you can pick a project easily from a shell, by taking advantage of a file name completion and other smart features in shell like this.

If you need it to be even more quicker...

This plugin also opens a TCP port 39524 and waits for a connection. You can have the plugin open a project by simply sending a directory name that contains .ipr or pom.xml, or a name of those files. For example, on Unix, such script could be: 

#!/bin/sh
D=`dirname "$1"`
B=`basename "$1"`
abspath="`cd \"$D\" 2>/dev/null && pwd || echo \"$D\"`/$B"
echo $abspath | nc -q 1 localhost 39524

I haven't tried this on Windows, but I suppose telnet command would do.
